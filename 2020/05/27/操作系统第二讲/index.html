进程管理

进程状态和进程控制

进程：一个并发程序在一个处理机上的执行过程

进程三大块：code,data,PCB

程序两部分:code,data

1. 动态性
 2. 并发性
 3. 独立性
 4. 制约性
 5. 结构性

为什么要引入进程概念：

​ 刻画并发程序的特性，便于管理多道程序，管理处理机等资源

五状态进程

七状态进程

挂起：为了减少内存的压力

PCB

将管理一个进程需要的所有信息存放在PCB中。

进程控制

  1. 创建
  2. 调度
  3. 撤销
  4. 阻塞
  5. 唤醒
  6. 挂机
  7. 激活

调度算法评价

  1. 吞吐率
  2. 周转时间
  3. 平均周转时间
  4. 带权平均周转时间
  5. 响应时间

常见的调度算法

  1. FCFS
  2. RR（轮转）
  3. SJF（短者优先）
  4. HPN（优先级高者优先）
  5. HRRN（相应比高者优先）
