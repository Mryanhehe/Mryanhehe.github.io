An End-to-End Learning-based Cost Estimator
发表于 2021-11-21| 更新于 2021-11-22|数据库调优
阅读量:

本文是对An End-to-End Learning-based Cost Estimator的翻译和笔记

abstract

开销(工作的估计量)和基数(步骤执行返回的行数)估计对于查询优化器来说至关重要，它可以指导计划查询的选择。然而，传统的经验开销和基数估计技术因为无法正确捕获多表之间的相关性，从而不能得到高质量的预估。近期数据库社区展示 基于学习的基数估计比经验估计更有效，然而现存的基于学习的方法有如下缺点

  1. 聚焦估计技术，但是不能估计开销
  2. 要么过于沉重，要么难以表示复杂的结构，例如复杂的谓词

为了解决以上问题，提出了通过树状组织模型一种有效的端对端的基于学习的开销估计框架，在特征提取中同时考虑查询和物理操作。将这些特征嵌入到树状组织中，可以提高谓词匹配的泛化能力。因为枚举所有的字符串成本较高，我们设计了一种模式匹配方法，会选择模式来匹配字符，并用模式来向量化字符串。

introduction

查询调优对数据库系统至关重要，其目的是为SQL语句选择最优的查询计划。然而，最近研究显示经典的查询优化器经常会因为较差的开销和基数估计而产生次优化查询。原因如下

  1. 传统经验开销/cardinality估算技术不能捕获多个列的相关性，尤其是对大的table和column
  2. 开销模型需要由DBA来调优

近些年，社区试图利用机器学习模型来提高基数估计的性能。MSCN采用卷积神经网络来估算基数，然而，该技术仍有缺陷

  1. 仅仅估计了技术，没有估计开销
  2. average pooling的深度神经网络很难表示复杂结构。比如模型很难泛化到其他sql语句

设置基于机器学习的开销有四个挑战

  1. 需要设计一个端对端的模型来预估cost和cardinality
  2. 学习模型应该捕获查询计划的树形结构信息
  3. 如果我们不知道哪个字符串满足的字符模式，支持带有字符串值的谓词相当困难，比如“not like”。字符串过于稀疏，很难讲字符嵌入到模型中
  4. 模型应该具有较强的泛化能力去支持各种各样的sql语句

为了应对，我们通过神经网络提出了端对端的基于机器学习的开销预估模型，设计了一个树状模型，可以从每个子计划中学习到其表示，可以无缝地替代传统的成本估算器。树状模型也可以表示有数值和字符sql语句的复杂预测。

contributions

  1. 一个基于树状模型，设计了一个有效的基于机器学习的端对端的成本预估模型，可以同时预估cost和cardinality
  2. 提出了有效的特征提取和向量化技术，在特征提取中考虑了查询和物理操作。将这些特征嵌入到树状模型中，利用树状模型可以估计cost和candinality
  3. 对于带有字符串的谓词，还提出了一种有效的技术来对字符串编码，从而提高泛化性能。由于枚举所有的字符串代价太高，设置了一种模式匹配的方法，会选择匹配字符串的模式，通过该模式来向量话字符串
  4. 实验验证，该模型表现良好
